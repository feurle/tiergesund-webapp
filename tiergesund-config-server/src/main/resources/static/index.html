Spring Cloud Config Server

The Spring Cloud Config Server provides an HTTP resource-based API for external yaml configuration for all the Tiergesund Webapp

  • See the Spring Cloud Config Server reference manual for the REST API
  • All Tiergesund configuration files in GitHub

Microservices Configurations

All configuration yml files for the microservices are served within the Spring Cloud Config Server. Based on the deployments, two types of configurations are available: default (native on host) and docker.

  • application-default.yml - common configuration for all microservices
  • discovery-server.yml - discovery server

Configuration of the services deployed native on host

  • vets-service-default.yml
  • visits-service-default.yml
  • customers-service-default.yml
  • api-gateway-default.yml

Configuration of the services deployed on a Docker container

  • vets-service-docker.yml
  • visits-service-docker.yml
  • customers-service-docker.yml
  • api-gateway-docker.yml
